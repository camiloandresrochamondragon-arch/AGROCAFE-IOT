El prototipo físico

Así se ve la maqueta

Construida sobre una caja plástica con tierra y sustrato orgánico, integra el cultivo, los sensores y el sistema de riego por goteo en un solo módulo.

El problema

¿Por qué riego inteligente?

En el Eje Cafetero, los sistemas de riego tradicionales operan por horarios fijos o de forma manual, sin considerar las condiciones reales del suelo ni el clima del momento. Esto genera dos problemas críticos:

💧

Desperdicio hídrico

Se riega aunque el suelo ya tenga humedad suficiente, malgastando agua de forma innecesaria.

🌡️

Estrés hídrico

En picos de temperatura o baja humedad, el riego manual llega tarde y afecta la calidad del grano.

🤖

Nuestra solución

Un sistema IoT que mide en tiempo real, riega de forma autónoma según el suelo y registra cada lectura para analizarla.

Cómo funciona

Arquitectura del sistema

Un pipeline completo desde la captura física del dato hasta la visualización analítica, en cinco capas funcionales.

1

Captura IoT Activo

Sensores conectados al Arduino Nano: BME280 (temperatura, humedad y presión), LDR (luz) y sensor resistivo de humedad de suelo. Lee cada 2 segundos y envía JSON por puerto serial.

2

Dashboard en tiempo real Activo

Script en Python (Matplotlib + PySerial) que recibe el JSON, grafica las variables y permite el control manual de la bomba (ON / OFF / AUTO).

3

Almacenamiento CSV Activo

Cada lectura se exporta a un archivo CSV con marca de tiempo. Es el archivo que alimenta esta plataforma web con los datos históricos del prototipo.

4

Base de datos + ETL En proceso

Carga del CSV en una base de datos relacional mediante un proceso ETL. Tablas: Sensores, Mediciones, Cultivos y Alertas.

5

Análisis en Power BI En proceso

Dashboard analítico conectado al Data Warehouse. KPIs: consumo de agua, eventos de riego, alertas por suelo seco y correlación temperatura-humedad.

Hardware

Componentes de la maqueta

Cada elemento cumple un rol dentro del ciclo IoT: percibir, decidir y actuar sobre el cultivo.

Componente Referencia Función en el sistema
Microcontrolador Arduino Nano Procesa las lecturas, ejecuta la lógica de riego y envía datos JSON por serial a 9600 baud.
Sensor ambiental BME280 (I2C) Mide temperatura del aire, humedad relativa y presión atmosférica. Bus I2C en pines A4/A5.
Sensor de luz LDR — pin A0 Fotorresistencia que infiere la radiación solar para estimar la evapotranspiración.
Sensor de suelo Resistivo — pin A1 Mide la humedad del suelo (RAW ~20 seco a ~380 saturado), convertida a porcentaje 0–100%.
Driver de potencia L298N Controla la mini bomba de agua. ENA en D9 (PWM), IN1 en D10, IN2 en D11.
Actuador Mini bomba + mangueras Riego por goteo con dos ramales y cuatro goteros sobre el sustrato. Alimentación 12V.
Lógica de control

Modos de operación del riego

El firmware del Arduino implementa tres modos. El sistema decide por sí mismo cuándo regar (edge computing), pero permite override manual desde el dashboard.

AUTO

Automático

Si la humedad de suelo baja del umbral seco, activa la bomba por máximo 5 segundos. Al alcanzar el umbral húmedo, la detiene. Se repite en cada ciclo de lectura.

MANUAL ON

Riego manual

Activa la bomba sin restricción de tiempo desde el dashboard. El sensor de suelo no la detiene; persiste hasta recibir la orden de apagado.

MANUAL OFF

Detener

Detiene la bomba de inmediato y regresa el sistema al modo automático para retomar el control por sensores.

Big Data

Las 5 V del proyecto

El sistema cumple con los cinco criterios de Big Data aplicados a la agricultura de precisión.

Volumen

Lecturas cada 2 s → hasta 43 200 registros por día y por lote.

Velocidad

Procesamiento en tiempo real, con el dashboard actualizando a 1 Hz.

Variedad

5 variables heterogéneas: física, química y óptica del cultivo.

Veracidad

JSON validado con control de errores en cada transmisión.

Valor

Acción automática de riego basada en datos reales del suelo.

Integración sensor

Conectar el ESP32 en tiempo real

En la siguiente versión, el ESP32 con Wi-Fi enviará los datos a este endpoint. Cada POST inserta una nueva fila en la base de datos y actualiza la vista automáticamente.
